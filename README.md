# Go Game

Интерактивная текстовая игра с элементами чата. Проект на Go с интеграцией мессенджера, системой сюжетов и сохранением прогресса.

## Особенности

* Чат‑бот для прохождения интерактивных историй
* Система сохранения прогресса пользователя
* Поддержка миграций БД
* Конфигурация через Viper
* Логирование действий
* Интеграция с внешним мессенджером (через MAX API)

## Технологии

* **Язык:** Go 1.21+
* **ORM:** GORM
* **Конфигурация:** Viper
* **HTTP‑фреймворк:** Gorilla Mux / Gin (опционально)
* **Логирование:** Zap / стандартный log
* **БД:** PostgreSQL / MySQL / MariaDB

## Структура проекта

```
go-game/
├── cmd/app/main.go                 # Точка входа с graceful shutdown
├── internal/
│   ├── config/config.go            # Конфигурация через viper
│   ├── models/                     # Модели данных
│   ├── repository/                 # Репозитории с GORM
│   ├── service/                    # Бизнес‑логика + сервис мессенджера
│   ├── handler/                    # HTTP обработчики
│   └── migration/                  # Миграции БД
├── pkg/
│   ├── messenger/                  # Клиент для MAX API с тестами
│   ├── db/                         # Подключение к БД
│   └── logger/                     # Логирование
├── interfaces/                     # Все интерфейсы приложения
├── migrations/                     # SQL миграции
└── configs/                       # Конфигурационные файлы
```

## Установка и запуск

### Предварительные требования

* Go 1.21+
* PostgreSQL / MySQL
* `make` (опционально, для удобства)

### Шаги установки

1. Клонируйте репозиторий:
```bash
git clone <URL-репозитория>
cd go-game
```
2. Установите зависимости:
```bash
go mod download
```
3. Настройте конфигурацию:
```bash
cp configs/config.example.yaml configs/config.yaml
# Отредактируйте configs/config.yaml под свои нужды
```
4. Запустите миграции БД:
```bash
go run cmd/migration/main.go
```
5. Запустите приложение:
```bash
go run cmd/app/main.go
```

## Конфигурация

Настройте `configs/config.yaml` под свою среду:

```yaml
server:
  port: 8080
  host: "localhost"
database:
  dialect: "postgres"
  dsn: "host=localhost user=go_game dbname=go_game sslmode=disable"
messenger:
  api_url: "https://api.max.com"
  token: "your-token-here"
logging:
  level: "info"
```

## Миграции БД

Миграции расположены в папке `migrations/`. Для применения:

```bash
go run cmd/migration/main.go
```

Доступные команды миграции:
* `up` — применить все миграции;
* `down` — откатить последнюю миграцию;
* `status` — показать статус миграций.

## Запуск в Docker (опционально)

1. Соберите образ:
```bash
docker build -t go-game .
```
2. Запустите контейнер:
```bash
docker run -p 8080:8080 go-game
```

## Тестирование

Запустите все тесты:
```bash
go test ./... -v
```

Запустите тесты для конкретного пакета:
```bash
go test ./internal/service -v
```

## API эндпоинты (если используется HTTP)

* `GET /health` — проверка работоспособности
* `GET /api/stories` — список доступных историй
* `POST /api/answers` — отправка ответа пользователя
* `GET /api/progress` — прогресс пользователя

## Использование чат‑бота

После запуска бота в мессенджере доступны команды:
* `/start` — начать новую игру;
* `/progress` — показать текущий прогресс;
* `/stories` — выбрать историю для прохождения.
